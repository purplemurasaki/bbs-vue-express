# アーキ要件

---

|                | ライブラリ    | ホスト先 | 備考                   |
| -------------- | ------------- | -------- | ---------------------- |
| フロントエンド | Vuejs         | EC2      | フロントバック同サーバ |
| バックエンド   | Express       | EC2      | フロントバック同サーバ |
| DB             | MySQL         | RDS      |                        |
| 画像           | -             | S3       | CloudFrontを前に挟む   |
| テスト         | Jest          | -        | バックエンドのUTのみ   |
| インフラ構築   | Terraform     | AWS      |                        |
| CICD           | GitHubActions | -        |                        |

- サーバは一台構成
- コスト重視
- EC2はpublic subnet、RDSはprivate subnet
- EC2は手動ではSSMでのみアクセス可能
- mainブランチにプッシュのタイミングでCIでテストとビルド、CDでデプロイ

# アプリ要件

---

- 掲示板アプリケーション
- 画面・機能構成
  - 投稿一覧
    - 1ページ当たり10件
    - 11件目以降はページングで表示
    - 投稿者、投稿内容、投稿日時が表示される
    - 最下部には削除ボタンがあり、押下するとその投稿を削除する
  - 投稿作成・修正
    - 投稿者、投稿内容を入力
    - 修正は投稿者以外も可能
    - 投稿内容に画像を含むことも可能

# 進め方

---

1. GitHubのリポジトリを作成
   - 以降の手順は本リポジトリにプッシュしていく
2. アーキの設計書をmd形式で作成
3. アプリ(テーブル設計書含む)の設計書をmd形式で作成
4. プロジェクトのルールを作成
5. フロントエンドのひな型を作成
6. バックエンドのひな型を作成
7. CI/CDのひな型を作成
   - バックエンドのテスト自動化
   - フロントエンド、バックエンドのビルド
8. DML、DDLの実装
9. バックエンドを実装
10. フロントエンドを実装
11. Terraformでインフラ構築
12. CI/CDで自動デプロイまで構築

# その他

---

- 基本的な進め方はCursorの[ベストプラクティス](https://cursor.com/ja/blog/agent-best-practices)に沿って実行すること
- 想定している構成に開発のベストプラクティスに反する内容があれば、その部分を指摘、新しい構成の提案を実施すること
